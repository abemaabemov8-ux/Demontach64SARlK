# 📘 Инструкция по публикации на GitHub Pages

## Пошаговое руководство для начинающих

### Шаг 1: Создание аккаунта на GitHub

1. Перейдите на [github.com](https://github.com)
2. Нажмите **Sign up** (Регистрация)
3. Создайте бесплатный аккаунт

### Шаг 2: Создание репозитория

1. После входа нажмите кнопку **+** в правом верхнем углу
2. Выберите **New repository** (Новый репозиторий)
3. Заполните:
   - **Repository name:** `demolition-saratov` (или любое другое имя)
   - **Description:** "Сайт компании ДемонтажСтрой"
   - Выберите **Public** (Публичный)
   - ✅ Поставьте галочку **Add a README file**
4. Нажмите **Create repository**

### Шаг 3: Загрузка файлов на GitHub

#### Вариант А: Через веб-интерфейс (Простой способ)

1. На странице вашего репозитория нажмите **Add file** → **Upload files**
2. Перетащите **ВСЕ** файлы проекта в окно браузера
3. Дождитесь загрузки
4. В поле "Commit changes" напишите: `Первая версия сайта`
5. Нажмите **Commit changes**

#### Вариант Б: Через Git (Для продвинутых)

```bash
# В терминале, находясь в папке с проектом:

git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ВАШ_USERNAME/demolition-saratov.git
git push -u origin main
```

### Шаг 4: Настройка GitHub Pages

1. В вашем репозитории перейдите во вкладку **Settings** (⚙️ Настройки)
2. В левом меню найдите **Pages**
3. В разделе **Build and deployment**:
   - **Source:** выберите **GitHub Actions** (не "Deploy from a branch")
4. Сохраните (если есть кнопка Save)

### Шаг 5: Запуск автоматической публикации

1. Перейдите во вкладку **Actions** (⚡ Действия) вверху страницы
2. Вы увидите процесс сборки "Deploy to GitHub Pages"
3. Подождите 2-5 минут, пока появится зелёная галочка ✅
4. Если видите красный крестик ❌, нажмите на него и посмотрите ошибки

### Шаг 6: Получение ссылки на сайт

После успешного деплоя ваш сайт будет доступен по адресу:

```
https://ВАШ_USERNAME.github.io/demolition-saratov/
```

Замените `ВАШ_USERNAME` на ваш логин GitHub и `demolition-saratov` на имя репозитория.

## 🔧 Обновление сайта

После первой публикации, чтобы обновить сайт:

1. **Через веб-интерфейс:**
   - Откройте нужный файл на GitHub
   - Нажмите кнопку ✏️ (редактировать)
   - Внесите изменения
   - Нажмите **Commit changes**
   - Сайт обновится автоматически через 2-5 минут

2. **Через Git:**
   ```bash
   git add .
   git commit -m "Описание изменений"
   git push
   ```

## ❓ Часто задаваемые вопросы

### Сайт не открывается / показывает 404

**Решение:**
1. Подождите 5-10 минут после первой публикации
2. Проверьте, что в Settings → Pages выбран **GitHub Actions**
3. Убедитесь, что во вкладке Actions процесс завершился успешно (зелёная галочка)
4. Попробуйте открыть сайт в режиме инкогнито

### Навигация не работает (страницы не переключаются)

Это нормально! Сайт использует Hash-роутинг, ссылки будут выглядеть так:
- Главная: `https://username.github.io/repo-name/#/`
- Услуги: `https://username.github.io/repo-name/#/services`
- О компании: `https://username.github.io/repo-name/#/about`

### Как подключить свой домен?

1. Купите домен (например, на REG.RU или Timeweb)
2. В Settings → Pages найдите раздел **Custom domain**
3. Введите ваш домен (например, `demolition-saratov.ru`)
4. В настройках DNS вашего домена добавьте записи:
   ```
   A    185.199.108.153
   A    185.199.109.153
   A    185.199.110.153
   A    185.199.111.153
   ```
5. Или CNAME запись: `username.github.io`
